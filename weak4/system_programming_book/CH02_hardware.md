# 第二章：硬體基礎

## 2.1 電腦硬體架構概述

### 2.1.1 冯·诺依曼架構

現代大多數電腦都採用冯·诺依曼（Von Neumann）架構，包含以下五個主要組成部分：
- **運算單元（Arithmetic Logic Unit, ALU）**：執行算術與邏輯運算
- **控制單元（Control Unit）**：控制指令執行順序
- **記憶單元（Memory）**：儲存程式與資料
- **輸入裝置（Input）**：接收外部資料
- **輸出裝置（Output）**：輸出運算結果

### 2.1.2 硬體組織圖

```
┌─────────────────────────────────────────────────────────────┐
│                           CPU                               │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐     │
│  │   控制單元   │───▶│     ALU     │◀───▶│   暫存器    │     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
└─────────────────────────────────────────────────────────────┘
            │                      │
            ▼                      ▼
    ┌───────────────┐      ┌───────────────┐
    │     記憶體     │      │   I/O 裝置    │
    │  (RAM/ROM)    │      │ (磁碟/網路/..) │
    └───────────────┘      └───────────────┘
```

## 2.2 CPU 架構與指令集

### 2.2.1 CPU 基本組成

- **暫存器（Register）**：CPU 內部的高速記憶體
- **算術邏輯單元（ALU）**：執行運算
- **控制單元**：解碼並執行指令
- **快取記憶體（Cache）**：減少記憶體訪問延遲

### 2.2.2 常見指令集架構

#### x86/x64 架構
- Intel 與 AMD 處理器使用
- 複雜指令集（CISC）
- 變長指令設計
- 廣泛用於個人電腦與伺服器

#### ARM 架構
- 精簡指令集（RISC）
- 定長指令設計
- 低功耗特性
- 用於手機、嵌入式裝置、蘋果 M 系列晶片

#### RISC-V 架構
- 開放指令集架構
- 模組化設計
- 越來越受到關注

### 2.2.3 指令執行流程

1. **取址（Fetch）**：從記憶體取得下一條指令
2. **解碼（Decode）**：解析指令意義
3. **執行（Execute）**：ALU 執行運算
4. **存取記憶體（Memory）**：讀寫記憶體
5. **寫回（Write Back）**：將結果寫回暫存器

### 2.2.4 常見暫存器類型

| 暫存器 | 功能 | x86-64 範例 |
|--------|------|-------------|
| 通用暫存器 | 存放運算資料 | rax, rbx, rcx, rdx |
| 指標暫存器 | 記憶體位址 | rbp, rsp |
| 索引暫存器 | 字串/陣列操作 | rsi, rdi |
| 程式計數器 | 下一條指令位址 | rip |
| 旗標暫存器 | 運算結果狀態 | rflags |

## 2.3 記憶體階層架構

### 2.3.1 記憶體階層

```
┌─────────────────────────────────────────────┐
│  L1 Cache (Level 1)     ~KB    最快          │
├─────────────────────────────────────────────┤
│  L2 Cache (Level 2)     ~MB    快           │
├─────────────────────────────────────────────┤
│  L3 Cache (Level 3)     ~MB    中           │
├─────────────────────────────────────────────┤
│  主記憶體 (RAM)          ~GB    慢           │
├─────────────────────────────────────────────┤
│  secondary storage     ~TB    最慢          │
└─────────────────────────────────────────────┘
  速度快 ◀──────────────────▶ 容量大
```

### 2.3.2 各層級特性

| 層級 | 大小 | 延遲 | 位置 |
|------|------|------|------|
| L1 Cache | 32-64KB | 1-2 周期 | CPU 核心內 |
| L2 Cache | 256KB-1MB | 10-20 周期 | CPU 晶片上 |
| L3 Cache | 8-64MB | 30-50 周期 | CPU 晶片上 |
| RAM | GB 級 | 100+ 周期 | 主機板 |
| 磁碟 | TB 級 | 百萬周期 | 外部儲存 |

### 2.3.3 快取記憶體運作機制

- **時間局部性（Temporal Locality）**：最近使用的資料很可能再次被使用
- **空間局部性（Spatial Locality）**：鄰近的資料很可能被同時存取
- **快取行（Cache Line）**：最小的快取單位，通常 64 bytes
- **映射方式**：直接映射、組相聯映射、全關聯映射

### 2.3.4 記憶體管理單元（MMU）

- 負責虛擬位址到實體位址的轉換
- 支援分頁（Paging）與分段（Segmentation）
- 提供記憶體保護

## 2.4 輸入輸出裝置

### 2.4.1 I/O 裝置分類

- **儲存裝置**：硬碟、SSD、光碟、隨身碟
- **網路裝置**：網卡、無線網卡
- **顯示裝置**：螢幕、投影機
- **輸入裝置**：鍵盤、滑鼠、觸控板
- **音效裝置**：喇叭、麥克風

### 2.4.2 I/O 控制方式

#### 程式化 I/O（Polling）
- CPU 不斷檢查裝置狀態
- 簡單但浪費 CPU 資源

#### 中斷驅動 I/O（Interrupt-driven）
- 裝置完成操作後發出中斷通知 CPU
- 效率較高

#### 直接記憶體存取（DMA）
- DMA 控制器直接在 I/O 裝置與記憶體之間傳輸資料
- CPU 無需參與大資料傳輸
- 效率最高

### 2.4.3 I/O 效能指標

- **頻寬（Bandwidth）**：單位時間可傳輸的資料量
- **延遲（Latency）**：一次 I/O 操作的時間
- **IOPS**：每秒 I/O 操作次數

## 2.5 中斷與例外處理

### 2.5.1 中斷的類型

- **硬體中斷**：來自外部裝置（鍵盤、滑碟、網路卡等）
- **軟體中斷**：由程式觸發（系統呼叫、陷阱）
- **例外**：由 CPU 產生（除錯錯誤、頁面錯誤）

### 2.5.2 中斷處理流程

```
1. 裝置產生中斷信號
2. CPU 完成當前指令
3. 儲存當前狀態（程式計數器、旗標等）
4. 查詢中斷向量表
5. 跳轉到中斷處理常式
6. 執行中斷服務
7. 恢復狀態並返回
```

### 2.5.3 中斷向量表

中斷向量表（Interrupt Vector Table）存放各類型中斷的處理常式位址：
- 每個中斷類型對應一個處理常式
- 位置通常在固定記憶體位址
- 可動態修改以支援作業系統

### 2.5.4 例外（Exception）類型

| 類型 | 原因 | 範例 |
|------|------|------|
| 錯誤（Fault） | 可復原的錯誤 | 頁面錯誤（page fault） |
| 陷阱（Trap） | 有意的例外 | 系統呼叫、斷點 |
| 中止（Abort） | 嚴重錯誤 | 硬體故障 |

## 2.6 匯流排系統

### 2.6.1 常見匯流排

- **系統匯流排**：連接 CPU 與記憶體
- **PCI/PCIe**：擴充卡標準
- **USB**：周邊裝置連接
- **SATA/NVMe**：儲存裝置連接
- **I2C/SPI**：嵌入式裝置

### 2.6.2 匯流排特性比較

| 匯流排 | 頻寬 | 用途 |
|--------|------|------|
| PCIe 4.0 x16 | 32 GB/s | 顯示卡、加速卡 |
| NVMe SSD | 3-7 GB/s | 高速儲存 |
| SATA III | 600 MB/s | 傳統硬碟/SSD |
| USB 3.2 | 20 Gbps | 周邊裝置 |
