# 第三章：作業系統概述

## 3.1 作業系統的定義

### 3.1.1 什麼是作業系統

作業系統（Operating System，簡稱 OS）是一套管理硬體資源並提供服務給使用者與應用程式的系統軟體。它作為硬體與應用程式之間的橋樑，讓使用者與開發者无需直接面對硬體的複雜性。

### 3.1.2 作業系統的目標

- **效率最大化**：有效率地管理硬體資源
- **便利性**：提供簡化的介面讓使用者操作電腦
- **公平性**：確保多個程式公平共享資源
- **保護性**：隔離不同程式，防止未經授權的存取

## 3.2 作業系統的核心功能

### 3.2.1 程序管理

- **程序（Process）**：正在執行的程式實例
- **執行緒（Thread）**：程序內的執行單位
- **排程（Scheduling）**：決定哪個程序/執行緒獲得 CPU 時間
- **程序同步**：協調多個程序間的執行

### 3.2.2 記憶體管理

- **記憶體配置**：為程式分配記憶體空間
- **虛擬記憶體**：擴展可用記憶體位址空間
- **分頁/分段**：記憶體的管理機制
- **記憶體保護**：防止程式越界存取

### 3.2.3 檔案系統

- **檔案操作**：建立、讀寫、刪除檔案
- **目錄管理**：組織檔案結構
- **空間管理**：追蹤磁碟使用情況
- **存取控制**：保護檔案安全

### 3.2.4 設備管理

- **驅動程式**：與硬體裝置溝通的軟體
- **緩衝管理**：提升 I/O 效率
- **中斷處理**：響應硬體事件
- **設備抽象**：提供一致的介面

## 3.3 核心態與使用者態

### 3.3.1 權限等級

```
┌─────────────────────────┐
│     使用者態 (User)      │  低權限，無法直接訪問硬體
├─────────────────────────┤
│   系統呼叫介面 (API)     │  兩態分界線
├─────────────────────────┤
│     核心態 (Kernel)      │  高權限，可訪問所有硬體
├─────────────────────────┤
│        硬體             │
└─────────────────────────┘
```

### 3.3.2 核心態（Kernel Mode）

- 可執行 CPU 特權指令
- 可訪問所有記憶體位址
- 可直接控制所有硬體裝置
- 作業系統核心在此模式下執行

### 3.3.3 使用者態（User Mode）

- 只能執行非特權指令
- 受限于虛擬記憶體空間
- 透過系統呼叫與核心溝通
- 應用程式在此模式下執行

### 3.3.4 系統呼叫（System Call）

使用者態程式請求核心服務的介面：
- 檔案操作：open、read、write、close
- 程序控制：fork、exec、exit
- 記憶體配置：brk、mmap
- 設備通訊：ioctl、read、write

## 3.4 作業系統的發展歷史

### 3.4.1 早期階段

- **1940s-1950s**：無作業系統，程式直接控制硬體
- **1950s-1960s**：批次處理系統（Batch Processing）
- **1960s**：多元程式設計（Multiprogramming）

### 3.4.2 發展里程碑

- **1969 年**：Unix 誕生
- **1980s**：個人電腦作業系統（DOS、Mac OS）
- **1990s**：視窗化介面普及（Windows 95、Linux）
- **2000s**：行動裝置作業系統（iOS、Android）
- **2010s**：雲端作業系統與容器技術

### 3.4.3 主要作業系統家族

| 家族 | 代表作品 | 特性 |
|------|----------|------|
| Unix | Linux、macOS、FreeBSD | 穩定、網路能力強 |
| Windows | Windows 10/11 | 桌上型市場主導 |
| 嵌入式 | RTOS、FreeRTOS | 即時性、低資源 |

## 3.5 核心架構類型

### 3.5.1 單核心（Monolithic Kernel）

所有核心功能運行在同一個位址空間：
- 優點：執行效率高
- 缺點：錯誤可能導致整個系統崩潰
- 代表：Linux、FreeBSD

### 3.5.2 微核心（Microkernel）

只保留最基本的功能，其他服務運行在使用者空間：
- 優點：穩定性高、易擴展
- 缺點：執行效能較低
- 代表：Minix、QNX

### 3.5.3 混合核心（Hybrid Kernel）

結合單核心與微核心的優點：
- 代表：Windows NT、macOS（XNU）

## 3.6 行程與執行緒

### 3.6.1 程序

程序的組成：
- 程式碼（Code）
- 資料（Data）
- 堆疊（Stack）
- 堆積（Heap）
- 程序控制區塊（PCB）

### 3.6.2 執行緒

執行緒是程序內的執行單位：
- 共享程序的資源
- 獨立的執行堆疊
- 提高並發性

### 3.6.3 多程序與多執行緒比較

| 特性 | 多程序 | 多執行緒 |
|------|--------|----------|
| 資源共享 | 隔離 | 共享 |
| 建立成本 | 高 | 低 |
| 通訊方式 | IPC | 共享記憶體 |
| 穩定性 | 較高 | 一程序崩潰影響全部 |

## 3.7 總結

作業系統是電腦系統的核心組件，負責管理硬體資源並提供服務給上層軟體。理解核心態與使用者態的區別、系統呼叫的運作方式，以及程序的排程與管理，是學習系統程式的基礎。
