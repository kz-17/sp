# 第二章：系統程式的定義與分類

## 2.1 系統程式的定義

### 2.1.1 什麼是系統程式

系統程式（System Software）是介於硬體與應用程式之間的軟體層，主要負責管理硬體資源並提供執行環境給應用程式。與應用程式不同，系統程式的目標不是完成特定的業務功能，而是提供基礎設施服務。

### 2.1.2 系統程式的特性

- **基礎性**：提供其他軟體運作的基礎
- **依附性**：依賴特定硬體架構
- **高效性**：追求資源使用效率
- **透明性**：通常在使用者無感知的情況下運作

## 2.2 系統程式的分類

### 2.2.1 語言處理器

#### 編譯器（Compiler）
將高階語言程式翻譯為機器語言或中介語言的程式。
- 例子：GCC、Clang、MSVC
- 輸出：目的碼（Object Code）

#### 直譯器（Interpreter）
直接執行高階語言原始碼的程式，不需要事先編譯。
- 例子：Python、Ruby、JavaScript 直譯器
- 特性：執行時翻譯，啟動快但執行較慢

#### 組譯器（Assembler）
將組譯語言翻譯為機器語言的程式。
- 例子：MASM、NASM、GAS
- 特性：一對一對應組合語言指令

#### 編譯器與直譯器的比較

| 特性 | 編譯器 | 直譯器 |
|------|--------|--------|
| 執行速度 | 快 | 慢 |
| 啟動速度 | 慢 | 快 |
| 跨平台 | 需重新編譯 | 可攜性高 |
| 錯誤報告 | 編譯時檢查 | 執行時檢查 |

### 2.2.2 系統工具

#### 連結器（Linker）
將多個目的碼檔案合併為單一可執行檔的程式。
- 靜態連結：所有符號在連結時解析
- 動態連結：符號在執行時解析（.so、.dll）

#### 載入器（Loader）
將可執行檔載入記憶體並準備執行的程式。
- 職責：配置記憶體、解析符號、重定位

#### 偵錯器（Debugger）
幫助開發者找出程式錯誤的工具。
- 例子：GDB、LLDB、WinDbg
- 功能：中斷點、記憶體檢視、單步執行

#### 剖析器（Profiler）
分析程式效能，找出效能瓶頸的工具。
- 例子：gprof、Valgrind、perf

### 2.2.3 系統服務程式

#### 行程管理
- Shell：命令列介面（Bash、PowerShell）
- 行程監控工具：ps、top、Task Manager

#### 檔案系統
- 檔案系統格式化工具
- 磁碟管理工具

#### 網路服務
- 網卡驅動程式
- 協定堆疊

### 2.2.4 作業系統核心

#### 核心功能
- 程序排程
- 記憶體管理
- 檔案系統
- 設備驅動

#### 核心類型
- 單核心（Monolithic Kernel）
- 微核心（Microkernel）
- 混合核心（Hybrid Kernel）

## 2.3 系統程式與應用程式的比較

| 層面 | 系統程式 | 應用程式 |
|------|----------|----------|
| 目標 | 提供執行環境 | 解決特定問題 |
| 執行層級 | 通常在核心態 | 使用者態 |
| 硬體相依性 | 高 | 低 |
| 開發難度 | 高 | 中等 |
| 穩定性要求 | 極高 | 中等 |

## 2.5 總結

系統程式是電腦系統中不可或缺的基础软件，了解其定義與分類有助於理解整個軟體堆疊的運作原理，為後續學習程序管理、記憶體管理等主題打下基礎。
